## 🚀 Como Contribuir

### 1. Reportar Bugs

Se encontrou um bug:
- Verifique se já não foi reportado
- Descreva o problema em detalhes
- Inclua passos para reproduzir
- Mencione sua versão do software

**Template de Issue:**
```
**Descrição do Bug:**
[Descreva o problema]

**Passos para Reproduzir:**
1. ...
2. ...
3. ...

**Comportamento Esperado:**
[Descreva o que deveria acontecer]

**Comportamento Atual:**
[Descreva o que realmente acontece]

**Ambiente:**
- OS: [Windows/Mac/Linux]
- Versão: [...]
```

### 2. Sugerir Melhorias

Sugestões são bem-vindas! Descreva:
- O problema que a melhoria resolveria
- Sua solução proposta
- Exemplos de como seria usada

### 3. Enviar Código

#### Configuração Inicial
```bash
# Clone o repositório
git clone <URL-do-repositorio>
cd PROJETOS\ OSF\ DIGITAL\ ACADEMY

# Crie uma branch para sua feature
git checkout -b feature/sua-feature
```

#### Fazer Mudanças
1. Faça mudanças incrementais e bem focadas
2. Teste suas alterações
3. Documente mudanças significativas

#### Commit
```bash
# Use mensagens de commit descritivas
git commit -m "Adiciona nova seção de testes de API"
git commit -m "Corrige typo na documentação"
git commit -m "Atualiza imagens das ferramentas"
```

#### Push e Pull Request
```bash
# Push para sua branch
git push origin feature/sua-feature
```

**Template de PR:**
```
## Descrição
[Descreva sua mudança]

## Tipo de Mudança
- [ ] 🐛 Bug fix
- [ ] ✨ Nova feature
- [ ] 📝 Documentação
- [ ] 🎨 Ajuste visual
- [ ] ♻️ Refatoração

## Checklist
- [ ] Código testado
- [ ] Documentação atualizada
- [ ] Mensagens de commit claras
- [ ] Sem conflitos com main

## Screenshots (se aplicável)
[Adicione screenshots se relevante]
```

---

## 📝 Diretrizes de Commit

Use commits descritivos:

```
✨ feat: Adiciona novo módulo de testes de API
🐛 fix: Corrige inconsistência na documentação
📝 docs: Atualiza README com novas imagens
🎨 style: Melhora formatação das imagens SVG
♻️ refactor: Reorganiza seções do README
🧪 test: Adiciona casos de teste para nova feature
```

---

## 🎨 Padrões de Conteúdo

### Para Documentação
- Use Markdown claro e bem estruturado
- Adicione exemplos práticos
- Inclua links para referências
- Mantenha atualizado com mudanças

### Para Imagens
- Use formas SVG quando possível (melhor para controle de versão)
- Mantenha proporções adequadas
- Garanta boa legibilidade
- Adicione descrições alt-text

### Para Código Excel
- Use nomes descritivos para abas
- Documente fórmulas complexas
- Mantenha organização por tópicos

---

## 🔄 Processo de Revisão

1. **Submissão**: Envie PR com descrição clara
2. **Análise**: Equipe revisará suas mudanças
3. **Feedback**: Podem solicitar ajustes
4. **Aprovação**: Após aprovação, PR é mergeado
5. **Publicação**: Mudanças aparecem na main

---

## ⚠️ Diretrizes de Qualidade

- ✅ Sem duplicação de informação
- ✅ Consistente com padrões existentes
- ✅ Testado antes de enviar
- ✅ Sem erros ortográficos
